# Промпт для Claude Design — редизайн карточки контрагента

> Скопируй этот файл целиком в Claude Design (или другой UI-генератор).
> Два примера JSON в конце — реальные ответы API с production.

---

## Контекст продукта

**goslog.art** — B2B SaaS для проверки экспедиторов и перевозчиков по 140-ФЗ. Получатель — логист, закупщик или юрист, который решает: **работать с этим контрагентом или нет**. Каждая секунда его внимания дорогая.

Сейчас у нас огромный объём данных по компании (см. JSON ниже): полный «Максимальный» тариф DaData (capital, licenses, managers с ИНН, contacts, authorities, documents, finance_history) + DataNewton (санкции, негативные списки) + risk_engine (детерминированный score 0-100) + **AI executive summary** (сгенерирован gpt-4.1-mini, signal-first, с structured risks/anomalies, каждый риск с evidence).

Старая карточка (`frontend/src/pages/CounterpartyReport.tsx`, ~2000 строк) показывает данные построчно как реестр — это **не работает**. Юзер тонет в полях.

## Цель редизайна

Превратить карточку из **«реестровой проверки»** в **«supplier intelligence profile»**. Премиальный, плотный, signal-first интерфейс уровня Bloomberg Terminal / Pitchbook — для профессионалов.

## Жёсткие принципы

### ✅ Хорошо
1. **Signal-first**. Сверху — то что сразу даёт ответ "работать или нет". Внизу — детали для тех кому надо.
2. **AI executive summary в фокусе** — это главная новая ценность продукта. Должно выглядеть как «короткий отчёт аналитика», а не «AI generated text».
3. **Каждый риск/аномалия с evidence** — ссылка на конкретные данные. Никаких висящих лейблов "Высокий риск" без объяснения.
4. **Dense но читаемо** — много полезной информации на единицу площади, но без визуального шума.
5. **Verdict в одну строку наверху** (зелёный/жёлтый/красный + 1 фраза итога) — для случая "у меня 5 секунд посмотреть".
6. **Иерархия по убыванию важности** — risks выше чем lookups, AI summary выше чем reference data.
7. **Compliance flags как badges** в верхнем блоке — «Адрес недостоверен», «Дисквалификация руководителя», «В санкциях» — это убийцы сделки.

### ❌ Плохо (НЕ делать)
1. **Не reestr-style**. Не таблица с 100 полями подряд.
2. **Не дамп всех полей DaData**. Если поле не несёт сигнала для решения — спрятать в раскрывающуюся секцию или вообще не показывать.
3. **Не "AI assistant chat"**. Никаких "Спросите что угодно", никаких конверсационных UI.
4. **Не длинные AI essays**. Summary — 3-5 строк, не больше.
5. **Не emoji-heavy**. Профессиональный дизайн без 🎯🚀⚡.
6. **Не show all the data**. Хороший дизайн скрывает столько же, сколько показывает.

## Технический стек

- **React 18 + Vite + TypeScript**
- **shadcn/ui + Radix UI** — есть всё: Card, Badge, Tabs, Collapsible, Tooltip, Dialog
- **Lucide-react** для иконок
- **Inline styles + CSS variables** в текущем проекте (`--gl-ink`, `--gl-indigo`, `--gl-green-ink`, `--gl-muted` и т.д.)
- **Tailwind** — поддерживается, но используется выборочно

Желательно: использовать существующие CSS variables проекта (`--gl-*`) если возможно, чтобы редизайн вписался в общий стиль приложения. Можно посмотреть `frontend/src/pages/OneCIntegration.tsx` для примера текущего стиля.

## API контракт

Endpoint: `GET /api/v1/companies/{inn}` (новый, frozen contract).
Authorization: `X-API-Key: glsk_live_...`

Pydantic-схема: см. `app/api/v1/schemas/company.py`. Полный список полей — в JSON-примерах ниже.

**Важные nullable-блоки** (всегда чекать `null`):
- `ai` — может быть `null` если AI summary не сгенерирован
- `head` — может быть `null` для странных кейсов
- `licenses`, `managers`, `founders`, `finance_history`, `sites` — могут быть `[]`
- `compliance.head_disqualified` и др. — bool (false если не флагнуто)

## Что должен вернуть Claude Design

1. **Один React-компонент** `CounterpartyProfile.tsx` (или несколько связанных) — целиком функционирующая карточка под этот JSON.
2. Без лишних зависимостей вне shadcn/ui + lucide-react.
3. С реальной обработкой всех null/empty состояний (ai=null, licenses=[], managers с одним элементом и т.д.).
4. Адаптивный (desktop-first, но не должно ломаться на 1024px).
5. Тёмная и светлая темы поддерживаются по `--gl-*` переменным проекта (если есть time — иначе только светлая).

## Что НЕ нужно

- Бэкенд код, миграции, API клиенты — всё это уже есть.
- Аутентификация, navigation, sidebar — это вне scope карточки.
- Чат-бот / ассистент / AI-конверсация — мы не идём в эту сторону.
- Графики помимо mini-chart финансов — никаких дашбордов.

# Reference Topics — шпаргалка по КД 3.1 для Konvey

Краткий справочник доменных понятий, нужных для разработки Konvey.
**Не учебник** — а индекс, чтобы найти, где искать детали.

---

## Архитектура обмена через EnterpriseData (КД 3.1)

```
КОНФИГУРАЦИЯ-ИСТОЧНИК (УТ 11.5)
    │
    │ [BSL-модуль с правилами выгрузки]   ← Konvey генерирует
    ▼
EnterpriseData XML (стандарт, описан XSD)
    │
    │ [BSL-модуль с правилами загрузки]   ← Konvey генерирует
    ▼
КОНФИГУРАЦИЯ-ПРИЁМНИК (БП 3.0)
```

Konvey генерирует **два BSL-модуля** — по одному на каждую сторону обмена.
Имя модуля: `МенеджерОбменаЧерезУниверсальныйФормат` (общий модуль конфигурации).

---

## Категории правил (объекты в BSL-модуле)

| Сокращение | Расшифровка | Что описывает |
|---|---|---|
| **ПКО** | Правило Конвертации Объекта | соответствие «1С-объект ↔ EnterpriseData-объект» (Document ↔ Документ.X) |
| **ПКС** | Правило Конвертации Свойства | соответствие реквизита (поля) внутри ПКО |
| **ПКТЧ** | Правило Конвертации Табличной Части | для документов с табличками |
| **ПОД** | Правило Обработки Данных | какие объекты выгружать, фильтры |
| **ПКПД** | Правило Конвертации Предопределённых Данных | для перечислений и предопределённых элементов справочников |

## Структура XSD EnterpriseData

Корневой namespace: `http://v8.1c.ru/edi/edi_stnd/EnterpriseData/<version>` (например, `/1.8`).

Основные паттерны:
- `xs:complexType` для каждого объекта (Документ, Справочник)
- `xs:complexContent + xs:extension base="ns1:Object"` — наследование от базового типа
- `xs:sequence + xs:element` — поля
- `xs:simpleType + xs:restriction + xs:enumeration` — перечисления
- `КлючевыеСвойства<Тип>` — composite key для ссылок (содержит GUID, Наименование, ИНН и т.д.)

Имена типов:
- `Документ.РеализацияТоваровУслуг`
- `Справочник.Контрагенты`
- `Перечисление.ВидыКонтрагентов`
- `КлючевыеСвойстваКонтрагент` (composite)
- `Документ.РеализацияТоваровУслуг.Товары` (tabular section)
- `Документ.РеализацияТоваровУслуг.Товары.Строка` (tabular row)

## Структура выгрузки XML конфигурации 1С

Корень: папка с `Configuration.xml`.

Подпапки (полный набор):
- `Catalogs/` — справочники
- `Documents/` — документы
- `InformationRegisters/` — регистры сведений
- `AccumulationRegisters/` — регистры накопления
- `AccountingRegisters/` — регистры бухгалтерии
- `Enums/` — перечисления
- `ChartsOfAccounts/` — планы счетов
- `ChartsOfCharacteristicTypes/` — планы видов характеристик
- `ChartsOfCalculationTypes/` — планы видов расчёта
- `BusinessProcesses/` — бизнес-процессы
- `Tasks/` — задачи
- `ExchangePlans/` — планы обмена
- `CommonModules/` — общие модули (нас интересуют только декларативно)
- `HTTPServices/`, `WebServices/`, `DataProcessors/`, `Reports/` — игнорируем для Konvey

Внутри каждой подпапки — `<ObjectName>.xml` + папка `<ObjectName>/Ext/` со связанными файлами (модули, формы, макеты).

Для Konvey важны только `<ObjectName>.xml` — там Attributes, TabularSections, EnumValues.

## Структура BSL-модуля «МенеджерОбменаЧерезУниверсальныйФормат»

Точка входа — процедура `ЗаполнитьПравилаКонвертации` или `ЗаполнитьПравилаКонвертацииОбъектов`.
Внутри регистрируются ПКО через вызовы:
```bsl
ПравилоКонвертации = ОбменДаннымиXDTOСервер.ИнициализироватьПравилоКонвертацииОбъекта(...);
ПравилоКонвертации.ИмяПКО = "Справочник_Контрагенты";
ПравилоКонвертации.ОбъектДанных = Метаданные.Справочники.Контрагенты;
ПравилоКонвертации.ОбъектФормата = "Справочник.Контрагенты";
ПравилоКонвертации.ПриОтправкеДанных = "ПКО_Справочник_Контрагенты_ПриОтправкеДанных";
...
```

Затем — ПКС внутри ПКО:
```bsl
ДобавитьПКС(СвойстваШапки, "ИНН", "ИНН");
ДобавитьПКС(СвойстваШапки, "Наименование", "Наименование");
```

И обработчики — отдельными процедурами в том же модуле:
```bsl
Процедура ПКО_Справочник_Контрагенты_ПриОтправкеДанных(...)
    // BSL-код пользовательской логики
КонецПроцедуры
```

## События ПКО (handlers)

| Событие | Когда вызывается | Зачем нужно |
|---|---|---|
| `ПередКонвертациейОбъекта` | до конвертации, до создания XDTO | проверки, фильтрация (`Отказ = Истина`) |
| `ПриОтправкеДанных` | при выгрузке (источник → ED) | заполнение XDTO-полей вручную |
| `ПриПолученииДанных` | при загрузке (ED → приёмник) | заполнение 1С-объекта из XDTO |
| `ПослеКонвертацииОбъекта` | после конвертации | пост-обработка, связи |
